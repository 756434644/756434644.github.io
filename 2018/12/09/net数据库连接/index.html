##数据库连接

  • 数据提供程序中包括：
    • Connection 连接对象
      • 用来和数据源进行连接
    • Command 命令对象
      • 用来从数据库中返回数据、修改数据或运行存储过程
    • DataReader 数据读取器
      • 提供了对数据库的快速只读、前向访问功能
    • DataAdapter 数据适配器
      • 是数据源和数据集 DataSet 对象交换数据的桥梁，它负责将数据库中的数据填充到 DataSet 对象中
  1. Connection属性和方法：
    • 构造方法：
      • SqlConnection()
      • SqlConnection(string connectionstring)
      • 注：如果需要重用Connection对象去使用不同的身份连接不同的数据库时，使用第一种方法则非常有效。
    • 方法：
      • open()
      • close()

  1. Command属性和方法：

    • 构造方法： 有两个
      • 一个是需要执行的SQL语句
      • 另一个是数据库连接对象

    • 属性：

      • CommandText： 获取或设置要对数据源执行的Transact—SQL语句或存储过程的名称。

      • CommandType： 获取或设置一个值，该值指示如何解释CommandText属性。

        cmd.CommandType = CommandType.Text

        它告诉.net接下来执行的是一个文本(text)、存储过程(StoredProcedure)还是表名称(TableDirect).

        而CommandType是一个枚举类型。有三个值：text、StoredProcedure、TableDirect用于表示SqlCommand对象CommandType的执行形式。

      • Connection： 获取或设置SqlCommand的实例使用的SqlConnection。

      • CommandTimeOut： 获取或设置在终止执行命令的尝试并生成错误之前的等待时间。
    • 方法：
      • ExecuteNonQuery(): 它的返回值类型为int型。多用于执行增加，删除，修改数据。返回受影响的行数。当select操作时，返回-1。
      • ExecuteReader(): 它的返回类型为SqlDataReader。此方法用于用户进行的查询操作,使用SqlDataReader对象的Read(),方法进行逐行读取。
      • ExecuteScaler():它的返回值类型Object类型。它返回的多为执行select查询。得到的返回结果为一个值的情况，比如使用count函数求表中记录个数或者使用sum函数求和等

  2. DataReader

    • 创建对象通过调用SqlCommand的ExecuteReader()方法返回DataReader()对象
    • 属性：
      • FieldCount属性：获取当前行中的字段数
    • 方法：
      • Read()方法：读取下一条记录，返回布尔值True表示此记录存在，False表示此记录为空
      • Close()方法：关闭SqlDataReader对象

  3. DataAdapter

    • 构造三种方式：

      • 通过连接字符串和查询语句

        SqlDataAdapter da=new SqlDataAdapter(strSQL,strconn);

      • 通过查询语句和SqlConnection对象来创建

        daOrders=new SqlDataAdapter(strSql,conn);

      • 通过SqlCommand对象来创建

  • SQL Server 数据库相关类都封装在System.Data.SqlClient 命名空间下，它主要包含4个类：

    • SqlConnection
    • SqlCommand
    • SqlDataAdapter
    • SqlDataReader

  • ADO.NET的数据访问策略：

    • 方式1：
      • Command及DataReader直接访问数据库并使用数据读取器（DataReader）来读取数据记录;
    • 方式2：
      • DataAdapter及DataSet,将数据记录缓存于数据集（Dataset）中

注:它们都要使用Connection及Command

方式1适合于只读数据，效率较高

方式2可以自动建立Command对象，适合于“离线”处理
